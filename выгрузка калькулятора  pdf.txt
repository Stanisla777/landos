У меня есть приложение калькулятора расчёта досрочного погашения ипотеки сделанное на vue 2. В этом калькуляторе есть блок с данными об ипотеке, блок с данными о погашении, блок с ипотечными каникулами, блок с графиком платежей с данными на основе расчёта того, что вбил пользователь. Задача реализовать выгрузку расчётов калькулятора. Нужно учесть, что дизайн калькулятора на странице отличается от дизайна итогового pdf. 
В pdf должна быть следующие блоки: 
шапка, которой нет на странице. Она состоит из двух колонок: Заголовка калькулятора и логотипа. Шапка должна бать на каждом листе pdf
блок, которого тоже нет  на странице, состоит из двух колоно: колонка с датой платежа и колонка с ссылкой на страницу с калькулятором
Блок данные об ипотеке
Блок данные о погашении
Блок с ипотечными каникулами
Блок с графиком платежей.
Надо так же учесть, что эти блоки отличаются от дизайна на странице.

Ворос, мне для выгрузки pdf сделать блок с новой вёрсткой, который не будет виден на странице у пользователя?
Блоки данные об ипотеке, данные о погашении не должны разрываться и если не помещаются на страницу полностью, должны переносится на новую. Но эти блоки, я думаю, и так поместяться на первой странице, а вот график платежей может быть длинным, допустимо 480 платежей ро месяцам, плюс в графике могут быть блоки с досрочным погашением, при чём к кажому месяцу и их может быть любое количество по месяцам, и ипотечные каникулы, но их может быть всего 4. 
Каждый платёж по месяцу - это отделый блок с несколькими рядами информации. И вот этот блок точно не должен разрываться и если не помещается на странице, то должна переносится на следующую. У каждой странице должна быть шапка.
Смогу я реализовать такую выгрузку? И повторю вопрос, мне нужно будет сделать для неё новую вёрстку согласно макету и указать плагину, который вы посоветуете использовать, какой блок формировать в pdf

html2canvas + jsPDF

Да, вы можете использовать flex и grid в разметке для PDF

Размеры и шрифты должны быть явно заданы (в px или pt)

Убедитесь, что ширина контейнера соответствует ширине листа PDF (например, A4 ≈ 595px при 72 DPI или 794px при 96 DPI).

Как тогда управлять разрывами страниц?
Поскольку html2canvas не делает разрывов сам:
Вы вручную разбиваете контент на «страницы» в JavaScript.
Для этого:
Рендерите каждый «неделимый» блок (например, один платёж) в отдельный div.
Последовательно добавляете их во временный контейнер.
После добавления каждого блока проверяете общую высоту контейнера.
Если превышает высоту страницы (например, 1123px для A4 при 96 DPI) — делаете снимок текущей страницы, очищаете контейнер, добавляете шапку и продолжаете.

Установка:
npm install jspdf html2canvas

Импорт в компоненте Vue 2:
import jsPDF from 'jspdf';
import html2canvas from 'html2canvas';

Простой экспорт одного блока (одна страница):
methods: {
  async exportToPDF() {
    const element = document.getElementById('pdf-export-content');

    const canvas = await html2canvas(element, {
      scale: 2, // улучшает качество
      useCORS: true, // если есть изображения с других доменов
    });

    const imgData = canvas.toDataURL('image/png');
    const pdf = new jsPDF('portrait', 'px', [canvas.width, canvas.height]);

    pdf.addImage(imgData, 'PNG', 0, 0, canvas.width, canvas.height);
    pdf.save('ipoteka-raschet.pdf');
  }
}
